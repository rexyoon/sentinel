# AI 회사 (작은 SI 팀) — 멀티 에이전트 시스템

## 프로젝트 개요
고객의 프로젝트 의뢰 한 줄을 받아, 역할을 맡은 여러 AI 에이전트(기획자·개발자·디자이너 등)가
협업해 소프트웨어 산출물(요구사항 정의 → 코드)을 만들어내는 멀티 에이전트 시스템.
"나만의 AI 회사"의 실체. **학습이 주 목적이다.**
(이전 프로젝트 Sentinel은 폐기했으나 git 히스토리 + GitHub rexyoon/sentinel 에 보존됨.)

## 핵심 개념
- **에이전트 1명 = 역할 지시문(system prompt) + 도구(tools) + LLM 루프**
  (LLM이 생각 → 도구 호출 결정 → 실행 → 결과 관찰 → 반복 → 완료)
- **회사 = 여러 에이전트 + 이들 사이에 일을 넘기는 오케스트레이터(매니저)**
- LLM: Claude (Anthropic API). 모델 ID·SDK 사용법은 claude-api 스킬 참고.

## 기술 스택 & 규칙
- Python 3.11+ (에이전트 생태계가 자연스러움).
- 표준 라이브러리 + `anthropic` SDK 위주. 과한 프레임워크(LangChain 등) 지양 — 학습 목적상
  루프·도구 호출을 직접 손으로 구현해 원리를 익힌다.
- 비밀키: `ANTHROPIC_API_KEY` 는 `.env` 로만 관리(.env.example 커밋). 하드코딩 금지.

## 작업 방식 (중요 — Sentinel에서 이어짐)
1. **코드는 사용자가 직접 타이핑한다.** Claude는 채팅에 코드 블록으로 출력하고, 실행·검증만 담당.
   설정/인프라 파일은 사용자가 "직접 해줘"라고 한 경우에만 Claude가 작성한다.
2. 구현 전 "무엇을/왜/어떻게" 3줄 계획을 먼저 보인다.
3. 핵심 로직엔 "왜 이렇게" 주석 + 작업 후 짧은 요약. 새 개념은 초심자 비유로 설명.
4. **한 번에 다 만들지 않는다.** 직원 1명(기획자)부터 제대로 → 협업 → 오케스트레이터.
5. git 작업(커밋/푸시)은 사용자가 직접 한다.

## 로드맵
- [ ] 1단계: 기획자(PM) 에이전트 — 의뢰 → 요구사항 정의. 에이전트 루프 + 도구(파일 저장) 학습.
- [ ] 2단계: 개발자 에이전트 추가 + 기획자→개발자 인수인계.
- [ ] 3단계: 오케스트레이터(매니저) — 누구에게 무슨 일을 시킬지 조율.
- [ ] 마지막: Claude API 키 연결해 실제 구동. (그 전까지는 골격·구조 위주)

## 하지 말 것
- 요청하지 않은 기능 추가(스코프 크리프).
- "키 없으면 못 한다"며 골격까지 미루기 — 키는 마지막, 구조는 먼저.
